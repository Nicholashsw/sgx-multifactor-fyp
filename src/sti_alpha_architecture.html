The map is everything. The build order is the map's purpose.

You wanted to see how it all fits — factor models, risk overlays, regime detection, ML, vol surfaces, IO linkages, options, attribution, execution, alt-data, the lot. Here it is. Eleven layers, thirty-eight modules. The teal modules are MVP-tier and ship first; amber are Year 2-3 expansions; grey are research/future-optionality and may never ship.

The architecture's job is twofold: (a) make every module's inputs and outputs explicit so any one can be built, swapped, or removed without breaking the others, and (b) make the build order visible so you don't accidentally try to ship Year 5 in Year 1 — the canonical failure mode of ambitious quant founders.

03 · CROSS-LAYER LINKAGES

Where Modules Meet

The non-obvious connections — where Module A's output is what Module B needs, often across non-adjacent layers. These are the contracts you must lock down first.

From To Interface (the contract) Why this linkage matters
Universe (L3) Factor Builder (L5) DataFrame[N×T] returns, fundamentals All factors require PIT exposures. If universe leaks future info, every downstream factor inherits the bias. Lock this contract first; everything else collapses without it.
Factor Builder (L5) Risk Model (L6) B (N×K) exposures matrix BARRA risk model uses the same exposures as alpha factors. Don't build two pipelines — one B matrix, two consumers. This is the single biggest architectural decision.
Macro Feed (L2) HMM Regime (L7) DataFrame[T×F] macro features SGD-USD vol, MAS slope, ACWI ret, VIX, DXY, Brent. Same panel feeds local projections (Phase 2). One macro panel, multiple consumers.
HMM Regime (L7) Alpha Combiner (L8) np.array[T×K_states] smoothed probs Regime probabilities modulate factor weights. Don't hard-code regimes; use posterior probs as continuous tilts so model failure is graceful.
FM Engine (L4) XGBoost Residual (L8) Series[N×T] residuals + B ML model fits residuals from linear FM, not raw returns. This is the difference between honest residual modelling and reinventing the linear factor model with a fancy name.
Alpha Combiner (L8) Optimiser (L9) α: np.array[N], Σ: np.array[N×N] Two inputs only — expected returns and covariance. If your optimiser needs more than this, you've leaked structure that should live one layer up.
Optimiser (L9) Cost Model (L10) target h*, current h, ADV Almgren-Chriss cost is f(trade size, ADV). Cost feeds back into the optimiser as penalty term — circular dependency that converges via SLSQP / OSQP.
Backtester (L10) Attribution (L11) Holdings panel, daily returns Brinson needs benchmark weights at each rebal date — keep this in the Universe abstraction so swapping benchmarks doesn't touch attribution code.
Risk Model (L6) Attribution (L11) B, F, factor returns f_t Factor-based attribution decomposes returns into factor contributions. Same risk model, two uses (forward: portfolio construction; backward: explain returns).
Local Projections (L7) Black-Litterman Views (L9) View vector P, Ω uncertainty Phase 2: Identified macro shocks become Bayesian views. Your independent macro research shock-identification work plugs directly here as the view-generator.
VRP / Options (L7) Stochastic Vol Risk (L6) Implied vol surface, σ_implied Future: Heston-calibrated implied vol enters the risk model as forward-looking covariance estimate. Marries your WorldQuant VRP work to the equity book.
Stat-Arb / Pairs (L5) Alpha Combiner (L8) Independent α stream Phase 2: pairs is a separate alpha source. Combine via diversification weight, not summation — uncorrelated alpha is the real prize.

05 · DECISION GATES

Don't Cross Without Earning It

Five gates that determine whether you're allowed to expand. Each one is binary. If the test fails, you go back and fix Phase N before touching Phase N+1.

G1
FYP Submitted, Bias Stat ∈ [0.85, 1.15]
Does the risk model actually forecast volatility correctly out-of-sample? If not, you have no foundation for portfolio construction.
UNLOCK → Phase 2A
G2
6 Months Live OOS, Sharpe > Benchmark
Does the strategy work in real life on real money? Paper trading via IBKR. If sim ≠ live, find out why.
UNLOCK → US universe
G3
First Outside Capital ($100k+)
Have you taken external capital, even a friends-and-family round, and run it for a year without a fraud / operational incident?
UNLOCK → Multi-asset
G4
$10m AUM, Audited Track Record
Audited GIPS-compliant performance, real LP letter cadence, regulatory licence. If you can't run $10m operationally, stop.
UNLOCK → Derivatives
G5
Capacity Saturated on Equities
Has the equities arm hit its capacity ceiling such that you genuinely need a second alpha stream? If not, don't diversify — concentrate.
UNLOCK → Execution alpha
∅
Failure Mode
If any gate fails, the answer is not "skip ahead." It's "fix the failed component or kill the arm." Founder discipline is the rarest input.
RETURN → previous phase
